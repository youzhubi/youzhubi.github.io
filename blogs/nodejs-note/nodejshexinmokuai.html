nodejs核心模块

fyj2023/05/01nodejs-notenodejs

核心模块是什么？核心模块，是node中自带的模块，可以在node中直接使用。window 是浏览器的宿主对象 node中是没有的global是node中的全局对象，作用类似于windowES标准下，全局对象的标准名应该是 globalThis。

# 1.process

  1. process模块用来表示和控制当前的node进程。
  2. 通过该对象可以获取进程的信息，或者对进程做各种操作。

如何使用

  1. process是一个全局变量，可以直接使用

  2. 有哪些属性和方法：

    process.exit([code])： 结束当前进程，终止node

    process.nextTick(callback[, …args])

    • 将函数插入到 tick队列中
    • tick队列中的代码，会在下一次事件循环之前执行,会在微任务队列和宏任务队列中的任务之前执行
setTimeout(() => {
    console.log(1)//宏任务队列
})

queueMicrotask(() => {
    console.log(2)
})              //微任务队列

process.nextTick(() => {
    console.log(3)
})            // tick队列

console.log(4)//调用栈

// 4 3 2 1

# 2.path

表示的是路径 通过path可以用来获取各种路径 要使用path，需要先对其进行引入

const path = require("node:path")

方法：

path.resolve([...paths]) : 用来生成一个绝对路径

相对路径：./xxx ../xxx xxx

绝对路径：

  • 在计算机本地
    • c:\xxx windows系统
    • /User/XXX Linux
  • 在网络中
    • http://www.xxxx/...
    • https://www.xxx/...

如果直接调用resolve,则返回当前的工作目录

const path = require("node:path")
const result = path.resolve("./01_path.js")
console.log(result)
//d:\htmlCssJs学习\nodejs\01_path.js

注意，我们通过不同的方式执行代码时，它的工作目录是有可能发生变化的

通过终端直接调用 node .\01_path.js

返回结果是D:\htmlCssJs学习\nodejs\03_核心模块\01_path.js

如果将一个相对路径作为参数，则resolve会自动将其转换为绝对路径 此时根据工作目录的不同，它所产生的绝对路径也不同

const path = require("node:path")
const result = path.resolve("./hello.js")
console.log(result) //d:\htmlCssJs学习\nodejs\hello.js
//终端调用输出
//D:\htmlCssJs学习\nodejs\03_核心模块\hello.js

一般会将一个绝对路径作为第一个参数，一个相对路径作为第二个参数， 这样它就会自动计算出最终的路径

const path = require("node:path")
const result = path.resolve("D:\\htmlCssJs学习\\nodejs\\03_核心模块", "./hello.js")
console.log(result)
//D:\htmlCssJs学习\nodejs\03_核心模块\hello.js

// 最终形态
// 以后在使用路径时，尽量通过path.resolve()来生成路径
//确保了在不同的运行方式下路径是正确的，以及换了操作系统的情况下
 const result = path.resolve(__dirname, "./hello.js")

# 3.fs

fs (File System)

  1. fs用来帮助node来操作磁盘中的文件
  2. 文件操作也就是所谓的I/O，input output
  3. 使用fs模块，同样需要引入

###1.writeFile（异步写入）

参数配置：

fs.writeFile(file, data[, options], callback)

参数说明：

  • file 文件名
  • data 待写入的数据
  • options 选项设置 （可选）
  • callback 写入回调

返回值： undefine

// 导入fs模块
const fs = require("fs")
// 写入文件 文件如果不存在会帮我们自动创建
fs.writeFile("./座右铭.txt", "三人行，则必有我师焉", (err) => {
    // err 如果写入失败:错误对象 成功:null
    if (err) {
        console.log("写入失败")
        return
    }
    console.log("写入成功")
})

# 2.writeFileSync

//参数与writeFile 大致无差 除了没有回调函数
// 写入文件 文件如果不存在会帮我们自动创建
fs.writeFileSync("./data.txt", "datadatum")

注意： VS Code 中按 F5 运行代码时，默认使用的是项目根目录作为当前工作目录，而在使用 node 命令执行代码时，默认使用的是当前命令行所在的目录作为当前工作目录。可以使用__dirname来获取当前文件所在的目录，使用path 模块的 join 方法来将当前文件所在的目录和文件名组合成一个绝对路径。

# 3.readFileSync

// 工作目录是D:\nodejs 查找不到   终端可以
const fs = require("node:fs/promises")
const buf = fs.readFileSync("./hello.txt")
console.log(buf)
// readFileSync() 同步的读取文件的方法，会阻塞后边代码的执行
// 当我们通过fs模块读取磁盘中的文件时，读取到的数据总会以Buffer对象的形式返回
// Buffer是一个临时用来存储数据的缓冲区，
const buf = fs.readFileSync(path.resolve(__dirname, "./hello.txt"))
 console.log(buf.toString())//也可以指定编码 通常是utf-8

# 4.readFile

//异步的读取文件的方法
fs.readFile(
    path.resolve(__dirname, "./hello.txt"),
    (err, buffer) => {
        
        if (err) {
            console.log("出错了")
        } else {
            console.log(buffer.toString())
        }       
    }
)

    .then(buffer => {

    .catch(e => {
        console.log("出错了")
    })

async版本

    }
})()

# 7.mkdir() 创建目录

const fs = require("node:fs/promises")
const path = require("node:path")
/*
    mkdir可以接收一个 配置对象作为第二个参数，
        通过该对象可以对方法的功能进行配置
            recursive 默认值为false
                - 设置true以后，会自动创建不存在的上一级目录
*/
fs.mkdir(path.resolve(__dirname, "./hello/abc"), { recursive: true })
    .then(r => {
        console.log("操作成功~")
    })
    .catch(err => {
        console.log("创建失败", err)
    })

# 8.rmdir() 删除目录

/*
    rmdir可以接收一个 配置对象作为第二个参数，
        通过该对象可以对方法的功能进行配置
            recursive 默认值为false
                - 设置true以后，会删除里面包含的目录以及自身的
*/
fs.rmdir(path.resolve(__dirname, "./hello"), { recursive: true })
    .then(r => {
        console.log("删除成功")
})

# 9.rm() 删除文件(推荐)

fs.rm(path.resolve(__dirname, "./hello"), { recursive: true })
    .then(r => {
        console.log("删除成功")
})

# Buffer（缓冲器）

# 1.概念

  • Buffer 是一个类似于数组的 对象 ，用于表示固定长度的字节序列
  • Buffer 本质是一段内存空间，专门用来处理 二进制数据 。

# 2.特点

  • Buffer 大小固定且无法调整
  • Buffer 性能较好，可以直接对计算机内存进行操作
  • 每个元素的大小为 1 字节（byte）

# 3.使用

创建buffer

// 1.alloc   alloc分配
//创建了一个长度为 10 字节的 Buffer，相当于申请了 10 字节的内存空间，每个字节的值为 0
let buf = Buffer.alloc(10)  // 结果为 <Buffer 00 00 00 00 00 00 00 00 00 00>

// 2.allocUnsafe
//创建了一个长度为 10 字节的 Buffer，buffer 中可能存在旧的数据（它不会对旧数据进行归零）, 可能会影响执行结果，所以叫unsafe
let buf2 = Buffer.allocUnsafe(10)
// 3.from
//通过字符串创建 Buffer  转换过程中每一个字符都会转成Unicode中对应的数字 
let buf3 = Buffer.from("hello") // <Buffer 68 65 6c 6c 6f>
//通过数组创建 Buffer
let buf4 = Buffer.from([105, 108, 111, 118, 101, 121, 111, 117])

Buffer字串转换

// buffer 字符串的转换
let buf4 = Buffer.from([105, 108, 111, 118, 101, 121, 111, 117])

Buffer读写

//读取
console.log(buf_3[1]);
//修改
buf_3[1] = 97;
//查看字符串结果
console.log(buf_3.toString())

注意：

  1. 如果修改的数值超过 255 ，则超过 8 位数据会被舍弃
  2. 一个 utf-8 的字符 一般 占 3 个字节
